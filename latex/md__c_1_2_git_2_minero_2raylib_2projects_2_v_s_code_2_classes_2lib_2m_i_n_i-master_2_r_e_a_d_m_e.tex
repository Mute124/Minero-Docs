This is a tiny, header only C++ library for manipulating INI files.

It conforms to the following format\+:
\begin{DoxyItemize}
\item section and key names are case insensitive
\item whitespace around sections, keys and values is ignored
\item empty section and key names are allowed
\item keys that do not belong to a section are ignored
\item comments are lines where the first non-\/whitespace character is a semicolon ({\ttfamily ;})
\item trailing comments are allowed on section lines, but not key/value lines
\item every entry exists on a single line and multiline is not supported
\end{DoxyItemize}


\begin{DoxyCode}{0}
\DoxyCodeLine{[section]}
\DoxyCodeLine{key\ =\ value}

\end{DoxyCode}


Files are read on demand in one go, after which the data is kept in memory and is ready to be manipulated. Files are closed after read or write operations. This utility supports lazy writing, which only writes changes and updates and preserves custom formatting and comments. A lazy write invoked by a {\ttfamily write()} call will read the output file, find which changes have been made, and update the file accordingly. If you only need to generate files, use {\ttfamily generate()} instead.

Section and key order is preserved on read and write operations. Iterating through data will take the same order as the original file or the order in which keys were added to the structure.

This library operates with the {\ttfamily std\+::string} type to hold values and relies on your host environment for encoding. It should play nicely with UTF-\/8 but your mileage may vary.\hypertarget{md__c_1_2_git_2_minero_2raylib_2projects_2_v_s_code_2_classes_2lib_2m_i_n_i-master_2_r_e_a_d_m_e_autotoc_md528}{}\doxysection{\texorpdfstring{Installation}{Installation}}\label{md__c_1_2_git_2_minero_2raylib_2projects_2_v_s_code_2_classes_2lib_2m_i_n_i-master_2_r_e_a_d_m_e_autotoc_md528}

The difference between {\ttfamily \mbox{[}\mbox{]}} and {\ttfamily \doxylink{raylib_2src_2external_2_a_n_g_l_e_2_e_g_l_2eglext_8h_afc5e04fa58feb228bb06199ff10caa22}{get()}} operations is that {\ttfamily \mbox{[}\mbox{]}} returns a reference to {\bfseries{real}} data (that you may modify) and creates a new item automatically if it does not yet exist, whereas {\ttfamily \doxylink{raylib_2src_2external_2_a_n_g_l_e_2_e_g_l_2eglext_8h_afc5e04fa58feb228bb06199ff10caa22}{get()}} returns a {\bfseries{copy}} of the data and does not create new items in the structure. Use {\ttfamily has()} before doing any operations with {\ttfamily \mbox{[}\mbox{]}} if you wish to avoid altering the structure.

You may combine usage of {\ttfamily \mbox{[}\mbox{]}} and {\ttfamily \doxylink{raylib_2src_2external_2_a_n_g_l_e_2_e_g_l_2eglext_8h_afc5e04fa58feb228bb06199ff10caa22}{get()}}.

Section and key names are case insensitive and are stripped of leading and trailing whitespace. {\ttfamily ini\mbox{[}"{}section"{}\mbox{]}} is the same as {\ttfamily ini\mbox{[}"{}\+SECTION"{}\mbox{]}} is the same as {\ttfamily ini\mbox{[}"{}   s\+Ec\+Ti\+On   "{}\mbox{]}} and so on, and same for keys. Generated files always use lower case for section and key names. Writing to an existing file will preserve letter cases of the original file whenever those keys or sections already exists.\hypertarget{md__c_1_2_git_2_minero_2raylib_2projects_2_v_s_code_2_classes_2lib_2m_i_n_i-master_2_r_e_a_d_m_e_autotoc_md535}{}\doxysubsection{\texorpdfstring{Updating data}{Updating data}}\label{md__c_1_2_git_2_minero_2raylib_2projects_2_v_s_code_2_classes_2lib_2m_i_n_i-master_2_r_e_a_d_m_e_autotoc_md535}

Keep in mind that {\ttfamily \mbox{[}\mbox{]}} will always create a new item if the item does not already exist! You can use {\ttfamily has()} to check if an item exists before performing further operations. Remember that {\ttfamily \doxylink{raylib_2src_2external_2_a_n_g_l_e_2_e_g_l_2eglext_8h_afc5e04fa58feb228bb06199ff10caa22}{get()}} will return a copy of data, so you should {\bfseries{not}} be doing removes or updates to data with it!

Usage of the {\ttfamily \mbox{[}\mbox{]}} operator shouldn\textquotesingle{}t be a problem in most real-\/world cases where you\textquotesingle{}re doing lookups on known keys and you may not care if empty keys or sections get created. However -\/ if you have a situation where you do not want new items to be added to the structure, either use {\ttfamily \doxylink{raylib_2src_2external_2_a_n_g_l_e_2_e_g_l_2eglext_8h_afc5e04fa58feb228bb06199ff10caa22}{get()}} to retreive items, or if you don\textquotesingle{}t want to be working with copies of data, use {\ttfamily has()} before using the {\ttfamily \mbox{[}\mbox{]}} operator if you want to be on the safe side.
